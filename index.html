Introducción a Git y Control de Versiones

¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite el seguimiento de cambios en los archivos de un proyecto.

Control de Versiones

El control de versiones es una práctica que consiste en llevar un registro de los cambios realizados en el código fuente y otros archivos de un proyecto.

Beneficios de usar Git

  • Historial completo de cambios
  • Colaboración efectiva en equipos
  • Posibilidad de deshacer cambios
  • Ramas para desarrollo paralelo

Comandos Básicos de Git

        
            git init       # Inicia un nuevo repositorio
            git add        # Añade archivos al área de preparación
            git commit     # Guarda los cambios en el repositorio
            git push       # Envía los cambios al repositorio remoto
            git pull       # Obtiene los cambios del repositorio remoto
            git clone      # Clona un repositorio existente
